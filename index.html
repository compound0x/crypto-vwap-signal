Deploying on GitHub Pages — this single file is enough:
  1. Rename this file to index.html
  2. Create a new GitHub repository and upload it (drag & drop in the web UI works)
  3. Repo Settings → Pages → Source: Deploy from a branch → main / root → Save
  4. Open https://<username>.github.io/<repo>/ — HTTPS is automatic, which also unlocks browser notifications
Diagnosing "unable to fetch": open DevTools (F12) → Console / Network tab. HTTP 451 means the exchange geo-blocks your region — deploying will not fix that (requests come from your browser, not from GitHub); the venue auto-failover above is the fix. Failed to fetch on a file:// page usually means a browser or extension restriction — deploying, or serving locally with npx serve ., fixes that. This version logs the exact per-venue failure in the log panel, so the cause is visible without DevTools.
